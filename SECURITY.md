# Security Policy

## Scope and intended use

This repository is intended for **defensive security** and **authorized testing** only.
Do not use repository content for unauthorized activities.

## Reporting expectations

Please report suspected vulnerabilities responsibly and in good faith.
When possible, avoid public disclosure until maintainers have had time to assess and mitigate risk.

## How to report a vulnerability

Open a private security advisory (preferred) or create a private communication channel available through GitHub's security reporting features.

If private reporting is unavailable, open an issue with minimal detail and request secure follow-up.
Do not publish exploit details or sensitive data in public issues.

## Suggested report template

Use this format to submit a useful report:

- **Title**: Short vulnerability summary
- **Affected component(s)**: File(s), workflow(s), dependency, or area
- **Severity estimate**: Low / Medium / High / Critical
- **Description**: What is the issue and why it matters
- **Impact**: Confidentiality / integrity / availability impact
- **Reproduction steps**: Clear, minimal, authorized steps
- **Evidence**: Logs, screenshots, commit hashes, scan output
- **Suggested remediation**: Optional fix recommendation

## Safe handling guidance

- Share only the minimum data needed for reproduction.
- Redact secrets, credentials, tokens, and personal data.
- Confirm testing was authorized for the target environment.
